static 静态修饰符

写在成员属性之前, 该成员称为静态成员属性

静态成员属性, 是静态的, 保存在类的定义内存中. 不管对象创建与销毁, 它都存在.

a) 静态成员属性

​ i. 静态修饰符(static)可以跟public protected private一起修饰成员属性,也可以单独来修饰成员属性

​ ii. 静态成员属性 是作为该类对象的一个共有属性存在

​ iii. 静态成员属性的访问方式

​ 1) 在类的内部 self::静态成员属性

​ 2) 在类的外部 类名::静态成员属性

b) 静态成员方法

​ i. 静态修饰符(static)可以跟public protected private一起修饰成员方法

​ ii. 静态成员方法中只能操作静态成员, 绝对不能在静态成员方法中使用$this

​ iii. 静态成员方法的访问形式

​ 1) 在类的内部 self::方法名

​ 2) 在类的外部 类名::方法名

类内和类外的静态成员属性访问

class A
  {
      static public $abc = 'wangjs';    // 1.定义静态成员

      public function xxoo()
      { 
          echo self::$abc;     // 2.成员方法中访问
      }
  }

  echo A::$abc;     // 3.外部 通过 类名::$属性名 访问

  $obj = new A;
  echo $obj::$abc;  // 4.外部 通过 对象::$属性名 访问  (这样访问慢一点)

类内和类外的静态成员方法访问

class A
  {
      public $abc = 'wangjs';    

      static public function xxoo()   // 1.定义静态成员方法
      { 
          return 100;
      }

      public function test()
      {
          echo self::xxoo();        // 2.访问静态成员方法
      }

  }

  echo A::xxoo();     // 3.外部 通过 类名::方法名 访问

常量 const

在类的内部定义常量, 只能使用 const


  class A
  {
     const XXOO = 'wangjs';   // 1.定义常量

     function say()
     {
         echo self::XXOO;       // 2.成员方法中 访问常量
     }
  }

  echo A::XXOO;     // 3. 外部 类名::常量名

  $obj = new A;
  echo $obj::XXOO;  // 4. 外部 对象::常量名

const和define的区别

​ i. const可以在类中定义常量 但是define不可以

​ ii. define赋值的时候可以使用变量进行赋值, 在函数的内部可以使用, const是不可以的

​ iii. define可以在流程控制中定义常量(if swtich) ,const是不可以的

单例模式

单例设计模式, 使某个类的对象仅允许实例化一次

1.不允许脚本在类的外部使用new关键字实例化对象.操作就是将构造方法和析构方法私有化

2.需要在类的内部声明一个公有的静态成员方法, 必须调用此方法 以返回实例的一个引用

3.声明静态成员属性存储新产生的对象

4.私有的__clone()方法防止克隆对象


    class A
    {
         private $name;
         private $age;
         static private $obj;    // 1.定义 静态+私有 成员属性

         private function __construct($name, $age)  // 2.私有化 构造方法  之后就不能 new 了
         {
               $this->name = $name;
            $this->age = $age;
         }

         public static function getObj()      // 3.静态+公有, 获取一个对象
         {
              if ( empty(self::$obj) ) {
                 self::$obj = new A('wangjs', 19);  // 如果以前没有创建过对象,就创建一个保存起来
              } 
              return self::$obj;   // 返回类中保存的对象实例
         }

         //__clone 魔术方法进行私有化   防止在外部进行克隆操作
        private function __clone()
        {
            $this->brand = '奇瑞';
            $this->price = '13w';
        }

        //防止在对象被销毁的时候  进行实例化对象
        private function __destruct()
       {
           var_dump(new Car('奇瑞','15w'));
       }
    }

__autoload 自动加载

a) __autoload是一个魔术函数(不是一个魔术方法)

b) 当使用new 关键字实例化一个不存在的类的时候 ,继承一个不存在的类的时候自动触发

c) 接受一个参数 不存在的类名(字符串类型)

d) 作用就是自动加载类文件 (类的自动加载)

<?php
    function __autoload($className)
    {
        include './libs/'.$className.'.php';
    }

    $a = new A;
?>


//注册一个自动加载类函数
spl_autoload_register('xxoo');

序列化和反序列化

serialize(串行化或者是序列化) 和 unserialize(反串行化或者是反序列化)

serialize() 是一个函数.

作用是将一个数组或者是对象 转化成一个字符串 便于持久化保存

应用场景: 缓存 方便数据传输

unserialize() 是一个函数

作用是将字符串还原成原来的数据,可能是数组也可能是对象

魔术方法

__sleep 当使用serialize进行串行化对象的时候 自动执行

作用 是要筛选要保留的属性的名称.

注意点: 必须要返回一个数组.

__wakeup 当使用unserialize反串行化一个对象的时候,会自动触发该魔术方法

作用 就是为了初始化反串行化出来的对象.

<?php 
    class A
    {
        public $name;
        public $age;

        public function __construct($name, $age)
        {
            $this->name = $name;
            $this->age = $age;
        }

        public function __sleep()
        {
            return array('age');
        }

        public function __wakeup()
        {
            $this->age = 30;
            $this->name = 'qqqq';
        }
    }

    /*
        //串行化
        $a = new A('wjs',19);
        $se = serialize($a);
        file_put_contents('xxoo.txt',$se);
    */

    /*
        //反串行化
        $a = file_get_contents('xxoo.txt');
        $c = unserialize($a);
        var_dump($c);
    */

抽象类

a). 使用abstract来修饰 类或者成员方法

b). 修饰的类不能实例化对象, 修饰的成员方法没有方法体

c). 如果一个类的内部 包含一个抽象方法,那么该类就是一个抽象类,必须要使用abstract来修饰

d). 子类继承抽象类 需要将抽象类中的所有抽象方法完全实现才能实例化对象,如果只实现了部分,那么该类仍然是一个抽象类 ,必须使用abstract来修饰

<?php 

    abstract class A
    {
        public $name;
        abstract function say();
    }

    class B extends A
    {
        public function say()
        {
            return '我终于实现梦想了....';
        }
    }

    $b = new B;
    var_dump($b->say());

接口

a). 接口是一种特殊的类, 声明方式用interface关键字

b). 接口中只能存在两种类型的代码 常量 和 抽象方法(抽象方法无需使用abstract来修饰,默认全部都是抽象方法)

d). 子类在继承(实现)接口的时候 ,使用的关键字是 implements

e). 如果子类在实现接口的时候 ,将接口中所有的抽象方法完全实现,那么该类就是一个完整的设计,就能实例化对象,如果只实现部分方法,那么仍然是一个不完整的设计,必须要使用abstract来修饰

f). 接口的作用 限定方法的名称 统一标准

<?php 

    interface A
    {
        const XXOO = 100;

        public function say();
        public function eat();
    }

    abstract class B implements A
    {
        public function say()
        {
            return '你在说话吗???';
        }
    }

    class C extends B
    {
        public function eat()
        {
            return '你在吃饭吗???';
        }
    }

    $c = new C;

    var_dump($c->say());
    echo C::XXOO;

类型约束

限定函数或者是成员方法的参数类型

注意: 类型约束 只支持两种类型的限定 一种就是数组,另一种就是对象

<?php
    //数组
        $arr = array('name'=>'qqq','age'=>20);
        function xxoo(Array $arr)
        {
            return $arr;
        }
        xxoo($arr);


    //对象 
        class A
        {
            public $name='wjs';
        }

        function xxoo(A $a)
        {    
            echo $a->name;
        }

        $a = new A;
        xxoo($a);

多态性

同一个操作 针对不同的子类对象,将产生不同的结果

指在父类中定义的属性或行为被子类继承之后，可以具有不同的数据类型或表现出不同的行为. 这使得同一个属性或行为在父类及其各个子类中具有不同的语义。

<?php   
    //定义一个USB接口，让每个USB设略都遵守这个规范
    interface USB {
        public function run();
    }

    //扩展一个USB键盘设置，实现USB接口
    class Ukey implements USB {
        public function run() {         //按键盘的功能实现接口中的方法
            echo "运行USB键盘设备<br>";
        }
    }

    //扩展一个USB鼠标设置，实现USB接口
    class Umouse implements USB {  
        public function run() {       //按鼠标的功能实现接口中的方法
            echo "运行USB鼠标设备<br>";
        }
    }

    //扩展一个USB存储设置，实现USB接口
    class Ustore implements USB {
        public function run() {         //按存储的功能实现接口中的方法
            echo "运行USB存储设备<br>";
        }
    }

    //声明一个电脑类， 去使用USB设置
    class Computer {
        //电脑类中的一个方法可以应用任何一种USB设备
        public function useUSB($usb) {
            $usb -> run();
        }
    }

    $computer = new Computer;                   //实例化一个电脑类的对象         

    $computer ->useUSB( new Ukey() );           //为电脑插入一个USB键盘设备，并运行
    $computer ->useUSB( new Umouse() );          //为电脑插入一个USB鼠标设备，并运行
    $computer ->useUSB( new Ustore() );         //为电脑插入一个USB存储设备，并运行

常见函数

1.class_exists() — 检查类是否已定义
2.get_class_methods() — 返回由类的方法名组成的数组
3.method_exists() — 检查类的方法是否存在
4.property_exists() — 检查对象或类是否具有该属性
5.get_class() — 返回对象的类名
6.get_object_vars() — 返回由对象属性组成的关联数组
7.get_parent_class() — 返回对象或类的父类名
